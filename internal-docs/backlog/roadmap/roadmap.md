# TeamOrchestrator.com Roadmap

This roadmap tracks website evolution as a product surface for Team Orchestrator.

## Milestone W1: Platform Foundation
*Focus: Reliable publishing and structural consistency.*

- Stabilize Astro routing/layout patterns for marketing, blog, and docs.
- Finalize docs information architecture and section ownership.
- Establish cycle handoff discipline (`PLAN`, `handoff`, active backlog, seed prompt).
- Standardize content metadata requirements and URL conventions.

## Milestone W2: Documentation Product Experience
*Focus: Discoverability and maintainable technical content.*

- Expand docs navigation with section-level governance.
- Add search-index artifact generation and integration path.
- Define ingestion strategy for selected `projectathena/docs` material.
- Add docs versioning/archive policy for product transitions.

## Milestone W3: Content Operations And Growth
*Focus: Publishing velocity with quality controls.*

- Add editorial templates for blog posts, release notes, and technical updates.
- Add pre-publish QA checks (links, metadata, route integrity).
- Add release cadence process tied to Team Orchestrator milestones.
- Add conversion-oriented CTA instrumentation.

## Milestone W4: Production Maturity
*Focus: Measurable performance and trustworthy operations.*

- Add SEO baseline and social preview metadata strategy.
- Add analytics event model and dashboard-ready event taxonomy.
- Add deployment smoke checks and broken-link monitoring.
- Add backup/rollback process for content regressions.

## Suggested Enhancements

1. **Documentation Change Ledger:**
   Add a public changelog page for docs updates by date and category.

2. **Product Surface Mapping:**
   Maintain a map that links each website page to source-of-truth product docs.

3. **Reference API Sync Track:**
   Generate API reference content from `projectathena` contracts/artifacts on a scheduled cadence.

4. **Content Freshness SLAs:**
   Assign freshness targets per section (e.g., docs every 30 days, marketing every release).

5. **Search Quality Benchmarks:**
   Define relevance acceptance checks before shipping docs search UI.

6. **Governance Review Cadence:**
   Add monthly roadmap/content governance review with explicit owner rotation.
